Readme

Mein Projekt Payinator soll etwas ähnliches wie PayPal abbilden.
Allerdings soll bei meinem Projekt eine "Aufrunden bitte"-Funktionalität wie aus Supermärkten angeboten werden.
Leider konnte ich alle Funktionen in diesem Zusammenhang aus Zeitgründen nicht fertig stellen.

Ein Nutzer kann bei meinem Service meherere Accounts nutzen.
Z.B. einen Musikaccount mit dem er/sie nur Noten kauft und einen Rechnungsacount mit dem er/sie alle Rechnungen tilgt.
Besonders wichtig bei der Bezahlung war mir die Anonymität, weshalb ein Account seinen Butzer nicht kennt.

Es ist Leider nicht möglich sich als Company (Partnerprojekt Webshop) anzumelden,
da dies vom OAuth System nicht Unterstützt wird.
Deswegen wurde eine Company in die Datenbank geseedet. Das Funktioniert über den Scheduler einer EJB.

Für das Css Layout ist zu großen Teilen Spectre Css verantwortlich.

Funktionen des Payinators jetzt:
  • Accounts eines Nutzers anzeigen und filtern.
  • Payments eines Nutzers anzeigen (ein/ausgehend)
  • Ein neues Payment erstellen.
    Genutzt in Partnerprojekt
  • Nutzer durch Orphan removal Komplett löschen.
Funktionen im Hintergrund
  • User mit OAuth System einloggen. Daraus folgt, dass nutzer erst nach einmaligen Login in der Datenbank erscheinen.
    Genutztes Partnerprojekt
  • Datenupdate des Users beim Login
  • Link der User session mit anderem Projekt
  • Automatischer wechsel zwischen Session- und Tokenbasierender Authentifizierung
